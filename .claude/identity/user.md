# User Profile — Operator Preferences

This file captures the operator's communication style, decision patterns, and workflow preferences. Agents reference this to personalize interactions.

## Communication Style
- **Tone**: Direct, technical, no fluff. Get to the point.
- **Detail level**: Show me the "what" and "why", skip the obvious "how" unless I ask.
- **Format preference**: Bullet points over paragraphs. Tables for comparisons. Code blocks for anything technical.
- **Response length**: Concise by default. Expand only when the topic demands it.

## Decision Patterns
- Prefers seeing options ranked by cost-effectiveness
- Values practical implementation over theoretical elegance
- Wants to understand trade-offs before committing
- Approves incrementally — ship small, validate, iterate

## Domain Context
- Building a multi-agentic AI operating system
- Technical background — comfortable with code, architecture, DevOps
- Business context — solo operator, cost-sensitive, values automation ROI
- Platform: Windows desktop, eventual VPS migration

## Workflow Preferences
- Morning briefings preferred (6-7 AM schedule window)
- Batch approvals over one-at-a-time interruptions
- Visual dashboard over terminal output for monitoring
- Git-based artifact management

## Pet Peeves
- Don't ask permission for things already covered by the rules
- Don't repeat information I've already acknowledged
- Don't pad responses with disclaimers or caveats unless genuinely important
- Don't suggest tools/frameworks without explaining why they're better than what we have
