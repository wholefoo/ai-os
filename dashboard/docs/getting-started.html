Quick Start

Get AI OS running locally in under five minutes. This guide covers prerequisites, installation, configuration, and launching the dashboard.

Prerequisites

  • Node.js 20+ — LTS recommended. Verify with node -v
  • npm 10+ — Ships with Node.js 20
  • Git — For cloning the repository
  • Stripe account — Required for subscription management (test keys work for development)
  • AI API keys — At least one of: Anthropic (Claude), DeepSeek, or xAI (Grok)

Installation

Clone the repository and install dependencies:

git clone https://github.com/wholefoo/ai-os.git
cd ai-os
npm install

Configuration

Copy the example environment file and fill in your credentials:

cp .env.example .env

Open .env and configure the required variables:

Variable Required Description
PORT No Server port (default: 3000)
NODE_ENV No development or production
STRIPE_SECRET_KEY Yes From Stripe dashboard → Developers → API keys
STRIPE_WEBHOOK_SECRET Yes From Stripe dashboard → Webhooks → Signing secret
STRIPE_PRO_PRICE_ID Yes Stripe Price ID for the Pro plan ($49/mo)
STRIPE_ENTERPRISE_PRICE_ID Yes Stripe Price ID for the Enterprise plan ($199/mo)
ANTHROPIC_API_KEY For AI Claude API key for Opus 4.8 agents (all effort tiers)
API_TOKEN Prod Bearer token for API authentication
Tip: For local development, use Stripe test keys (prefix sk_test_). No real charges will be processed.

Launch

Start the server in development mode:

npm start
# Dashboard at http://localhost:3000

Or use the dev script with auto-restart:

npm run dev

First Steps

  1. Visit the landing page at http://localhost:3000 to see the public site
  2. Log in using the login modal (email + password)
  3. Subscribe via the Stripe checkout flow (use test card 4242 4242 4242 4242)
  4. Access the dashboard at /app — all 16+ views are now available
  5. Run an agent from the Agent Fleet view to verify your AI API keys work

  skills/        19 procedural skill files

  identity/      Soul, user preferences, personality
  pipelines/     Declarative YAML skill chains
  projects/      Per-project context overrides
.magent/
  vault/         Knowledge base (raw, wiki, outputs)
  state/         Persisted runtime state
  team.yaml      Agent roster and escalation paths
dashboard/
  index.html     Public landing page
  app.html       Authenticated dashboard
  css/           Landing, legal, docs, and dashboard styles
  js/            Landing and dashboard scripts
  docs/          Documentation pages
deploy/
  nginx.conf     Reverse proxy config
  install-vps.sh VPS provisioning script
server.js        Express + WebSocket backend
Note: The .magent/state/ directory is auto-created on first run and excluded from version control. Your runtime state persists between server restarts.
