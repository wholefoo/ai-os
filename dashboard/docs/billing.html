Billing & Subscriptions

AI OS uses Stripe for subscription management with two paid tiers. The dashboard is gated behind an active subscription via session-based access control.

Checkout Flow

The subscription flow uses Stripe Checkout for secure, PCI-compliant payment processing:

  1. User clicks a pricing CTA on the landing page
  2. The server creates a Stripe Checkout Session via GET /api/stripe/checkout?plan=pro
  3. User is redirected to Stripe's hosted payment page
  4. After successful payment, Stripe redirects to /api/stripe/success
  5. The server creates a session cookie and redirects to /app

Webhook Handling

Stripe sends webhook events to POST /api/stripe/webhook for lifecycle management:

EventAction
checkout.session.completedActivate subscription, create session
customer.subscription.updatedUpdate plan tier
customer.subscription.deletedRevoke dashboard access
invoice.payment_failedNotify user, enter grace period
Important: The webhook endpoint receives raw request bodies (not JSON parsed) because Stripe requires the raw body for signature verification via STRIPE_WEBHOOK_SECRET.

Session Management

Dashboard access is controlled by a session cookie:

  • Cookie name: ai-os-session
  • Type: HTTP-only (not accessible via JavaScript)
  • Expiry: 30 days
  • Content: Cryptographically random token (not email or PII)
  • Validation: Server-side lookup maps token to email + plan

Access Control Flow

  1. User requests /app
  2. Server reads ai-os-session cookie
  3. Looks up session in memory (token → email + plan + expiry)
  4. If valid session with active plan: serve app.html
  5. If no session, expired, or no plan: redirect to /login
