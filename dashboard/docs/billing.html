Step 1 — Create a Stripe Account

  1. Go to dashboard.stripe.com/register
  2. Sign up with your email
  3. Verify your email and complete onboarding

Step 2 — Get Your Secret Key

  1. Go to dashboard.stripe.com/apikeys
  2. You will see two keys:
    • Publishable key (pk_test_...) — not needed for AI OS
    • Secret key (sk_test_...) — click “Reveal test key” and copy it
  3. Paste into your .env:
STRIPE_SECRET_KEY=sk_test_XXXXXXXXXXXXXXXXXXXXXXXX
Test vs Live: Start with test keys (sk_test_...). When ready to accept real payments, toggle “Test mode” off in the Stripe dashboard and use live keys (sk_live_...).

Step 3 — Create Products and Prices

Create one Stripe product for each paid subscription tier. The Free tier has no Stripe product. The Founders tier uses a one-time Checkout session created dynamically by the server.

Starter Plan ($49/month)

  1. Go to dashboard.stripe.com/products
  2. Click + Add product
  3. Fill in:
    • Name: AI OS Starter
    • Description: 15 AI agents, 5 departments, 3 SEO audits/month, email support
  4. Under Price information:
    • Pricing model: Standard
    • Price: $49.00
    • Billing period: Monthly
  5. Click Save product
  6. On the product page, copy the Price ID (starts with price_)
  7. Paste into .env:
STRIPE_STARTER_PRICE_ID=price_XXXXXXXXXXXXXXXXXXXXXXXX

Pro Plan ($99/month)

  1. Click + Add product again
  2. Fill in:
    • Name: AI OS Pro
    • Description: All 51 AI agents, full Virtual HQ, 10 departments, SEO Agency, knowledge vault
  3. Price: $99.00 / Monthly
  4. Save, copy the Price ID, paste into .env:
STRIPE_PRO_PRICE_ID=price_XXXXXXXXXXXXXXXXXXXXXXXX

Business Plan ($497/month)

Founders License ($9,997 one-time)

No Stripe product needed. The Founders fee is handled as a dynamic one-time Stripe Checkout session created by the server. The amount ($9,997) is set in LICENSE_CONFIG in server.js. No separate Price ID is required. Limited to 100 spots.

Step 4 — Set Up the Webhook

The webhook lets Stripe notify your server when payments succeed, subscriptions renew, or customers cancel.

  1. Go to dashboard.stripe.com/webhooks
  2. Click + Add endpoint
  3. Endpoint URL: https://yourdomain.com/api/stripe/webhook
  4. Events to listen to — click “Select events” and add:
    • checkout.session.completed
    • customer.subscription.updated
    • customer.subscription.deleted
    • invoice.payment_succeeded
    • invoice.payment_failed
  5. Click Add endpoint
  6. On the webhook page, click Reveal under “Signing secret” and copy it (whsec_...)
  7. Paste into .env:
STRIPE_WEBHOOK_SECRET=whsec_XXXXXXXXXXXXXXXXXXXXXXXX
Important: The webhook endpoint receives raw request bodies (not JSON parsed) because Stripe requires the raw body for signature verification. This is already handled in server.js — the JSON body parser skips the /api/stripe/webhook route.

Step 7 — Go Live

  1. Complete Stripe account activation (business info, bank account for payouts)
  2. Toggle Test mode off in the Stripe dashboard
  3. Copy the live keys (sk_live_..., whsec_...)
  4. Create the same products/prices in live mode (Price IDs will be different)
  5. Update your .env with all live keys
  6. Update the webhook endpoint to use live mode
  7. Restart: sudo -u aios pm2 restart ai-os --update-env
Before going live: Test the full checkout flow end-to-end with test cards. Verify the webhook fires correctly by checking pm2 logs ai-os for Stripe event handling logs.

Checkout Flow

The subscription flow uses Stripe Checkout for secure, PCI-compliant payment processing:

  1. User clicks a pricing CTA on the landing page
  2. The server creates a Stripe Checkout Session via GET /api/stripe/checkout?plan=pro
  3. User is redirected to Stripe’s hosted payment page
  4. After successful payment, Stripe redirects to the success URL
  5. The server creates a session cookie and redirects to /app

Founders License Checkout Flow

  1. Customer selects the Founders plan on the pricing page
  2. The server creates a $9,997 one-time Stripe Checkout Session
  3. After payment, the customer is provisioned with lifetime Enterprise access
  4. All future platform updates are included at no additional cost

Webhook Events

EventAction
checkout.session.completedActivate subscription or franchise, create session
customer.subscription.updatedUpdate plan tier (starter ↔ pro ↔ business ↔ enterprise)
customer.subscription.deletedRevoke dashboard access
invoice.payment_succeededConfirm renewal, extend access
invoice.payment_failedNotify user, enter grace period

Session Management

  • Cookie name: ai-os-session
  • Type: HTTP-only, Secure (production), SameSite: Lax
  • Expiry: 30 days
  • Content: Cryptographically random token (UUID v4)
  • Validation: Server-side lookup maps token → email + plan + role + expiry
  • Fallback: Bearer token via Authorization header (for API clients)
